# Git Rules

## Golden Rule
One task = One commit = One feature branch

## Commit Message Format
`[type]: [short description]`

Allowed types:
- `feat:` — New feature
- `fix:` — Bug fix
- `test:` — Add/change test
- `refactor:` — Code change without behavior change
- `docs:` — Documentation change
- `chore:` — Maintenance tasks

## Before Commit
1. `uv run ruff check --fix .` — Fix linting issues
2. `uv run pytest` — All tests must pass
3. Secret check: No API keys in staged files
4. Then: `git commit -m "[type]: [description]"`

## Forbidden
CRITICAL: NEVER make any code modifications or git commits while on the 'main' branch. Your FIRST action for any task MUST BE: `git checkout -b <branch_name>`. The system will block any direct commits to main.
- Large commits with multiple scattered changes
- Commit messages like "fix", "update", "changes"
