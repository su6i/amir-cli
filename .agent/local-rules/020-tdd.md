# TDD Rules

## Mandatory Order (Never violate)
1. First write a test that FAILS
2. Write the minimum code necessary to pass the test
3. Refactor (if necessary)
4. Run `uv run pytest` — all tests must pass

## Test Rules
- Filename: `test_[module_name].py` in the `tests/` folder
- Function name: `test_[state]_[expected_behavior]()`
- Each test should test one thing (Single Responsibility)
- Use fixtures for repetitive setup

## Coverage
- At least 80% coverage to merge to main
- `uv run pytest --cov=src --cov-report=term-missing`

## Forbidden
- Never write implementation before the test
- Never skip a test without a documented reason
- Never put `pass` in a test
